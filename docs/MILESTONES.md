## Milestone 3: オプトイン・オプトアウト管理 🔲 3月目標

**ゴール**: 受信者が自分で配信登録（オプトイン）・配信停止（オプトアウト）できる。データを永続的に保持する

### 必要な機能
- [ ] オプトイン
  - [ ] 登録フォーム（公開ページ）— メールアドレス + 名前入力
  - [ ] ダブルオプトイン — 確認メール送信 → リンククリックで正式登録
  - [ ] 登録日時・ソース（フォーム / API / 手動）の記録
- [ ] オプトアウト
  - [ ] 配信停止リンク — メールフッターに自動挿入（既存: マーケティングフッター実装済み）
  - [ ] `List-Unsubscribe` ヘッダー自動付与（RFC 8058 / One-Click）
  - [ ] 配信停止ランディングページの改善（理由選択 + 確認画面）
  - [ ] オプトアウト日時・理由の記録
- [ ] データ管理
  - [ ] コンタクトのステータス管理（active / unsubscribed / bounced）
  - [ ] オプトイン・オプトアウト履歴テーブル（監査証跡）
  - [ ] 一括配信時にunsubscribed/bouncedを自動除外（既存: isUnsubscribedチェック実装済み）
  - [ ] Dashboard でのステータス別フィルタリング・集計

### 前提条件
- MS1 完了 ✅
- 既存の `unsubscribes` テーブル + `contacts.isUnsubscribed` フラグを拡張

### 完了条件
登録フォームからオプトイン → 確認メール → 配信開始。配信停止リンクからオプトアウト → 即座に配信対象から除外。全履歴がDBに保持される
